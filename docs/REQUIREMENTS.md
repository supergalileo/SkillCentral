## 1. 项目概述

### 1.1 项目背景

随着 AI Agent 工具（WorkBuddy、Claude Code、Cursor 等）的普及，用户往往需要管理多个 agent 的 skill。目前存在以下痛点：
- 安装了 skill 后容易忘记
- 更换 agent 工具时需要重新安装 skill
- skill 太多，缺乏统一的管理界面

SkillCentral 旨在解决上述问题，提供一个统一的 skill 管理平台。

### 1.2 项目目标

- 统一管理所有 agent 的 skill
- 支持一键启用/禁用 skill 到指定 agent
- 提供可视化的 skill 浏览、搜索、标签管理
- 支持 skill 的导入、导出、备份、恢复

### 1.3 技术栈

- **语言**：C++ 17
- **UI 框架**：Qt 6
- **数据库**：SQLite 3（通过 Qt SQL 模块）
- **构建工具**：CMake
- **版本管理**：Git

## 2. 功能需求

### 2.1 中央库管理

#### 2.1.1 中央库结构

```
D:\skillLibrary\
├── library.db              ← SQLite 数据库（元数据）
├── skills\                 ← 所有 skill 原始文件夹
│   ├── web-access\
│   │   ├── SKILL.md
│   │   └── (其他文件)
│   ├── huashu-design\
│   │   ├── SKILL.md
│   │   └── (其他文件)
│   └── ...
└── backups\               ← 备份文件存放目录
    └── backup_20260613.db
```

#### 2.1.2 数据库设计

**skills 表**
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER PRIMARY KEY | 自增 ID |
| name | TEXT | Skill 名称（文件夹名） |
| path | TEXT | 中央库中的相对路径 |
| description | TEXT | 从 SKILL.md 提取的简短描述 |
| frequency | INTEGER | 使用频率等级（1=常用, 2=一般, 3=很少, 4=废弃） |
| created_at | DATETIME | 导入时间 |
| updated_at | DATETIME | 最后修改时间 |
| last_used | DATETIME | 最后使用时间 |

**tags 表**
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER PRIMARY KEY | 自增 ID |
| name | TEXT UNIQUE | 标签名称 |

**skill_tags 表**（多对多关系）
| 字段 | 类型 | 说明 |
|------|------|------|
| skill_id | INTEGER | 外键 → skills.id |
| tag_id | INTEGER | 外键 → tags.id |

**agents 表**
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER PRIMARY KEY | 自增 ID |
| name | TEXT UNIQUE | Agent 名称 |
| path | TEXT | Skill 存放路径 |
| enabled | BOOLEAN | 是否启用 |
| icon | TEXT | 图标路径（可选） |

**skill_agents 表**（多对多关系）
| 字段 | 类型 | 说明 |
|------|------|------|
| skill_id | INTEGER | 外键 → skills.id |
| agent_id | INTEGER | 外键 → agents.id |
| enabled | BOOLEAN | 是否在该 agent 中启用 |

**settings 表**
| 字段 | 类型 | 说明 |
|------|------|------|
| key | TEXT PRIMARY KEY | 设置项名称 |
| value | TEXT | 设置值 |

#### 2.1.3 导入 Skill

**支持来源**：
1. 从本地文件夹导入（选择文件夹，复制到中央库）
2. 从 WorkBuddy 扫描导入（自动扫描 `~/.workbuddy/skills/`）
3. 从 GitHub 仓库地址导入（克隆或下载 ZIP）
4. 从 ZIP 包导入（解压后复制到中央库）
5. 拖拽导入（拖拽文件夹或 ZIP 到窗口）

**重复检测**：
- 导入时发现同名 skill 已存在，弹出对话框：
  - 「覆盖」— 替换原有 skill
  - 「重命名」— 命名为 `name_1`、`name_2`...
  - 「取消」— 不导入

#### 2.1.4 导出 Skill

- 选中一个或多个 skill
- 打包成 `.zip` 文件
- 保存到用户指定路径

#### 2.1.5 备份与恢复

**自动备份**：
- 设置中可配置自动备份频率（每天/每周/每月）
- 备份文件保存为 `backup_YYYYMMDD.db`（仅备份数据库，skills 文件夹由用户自行备份）

**手动备份**：
- 点击「备份」按钮，立即创建备份

**恢复**：
- 选择备份文件，恢复数据库到备份时的状态
- 恢复前提示：恢复将覆盖当前数据，确认是否继续？

#### 2.1.6 Skill 验证

- 定期检查所有 skill 文件夹是否包含 `SKILL.md`
- 缺失的 skill 在卡片上标红警告
- 点击「修复」按钮，可以手动指定 `SKILL.md` 位置或删除该 skill

### 2.2 Agent 管理

#### 2.2.1 Agent 配置

**预置 Agent**（可扩展）：
1. WorkBuddy
2. Claude Code
3. Codex
4. MimoCode
5. Trae
6. OpenCode

**配置项**：
- 名称（可修改）
- Skill 存放路径（如 `C:\Users\Dongmiao\.workbuddy\skills\`）
- 是否启用

**添加自定义 Agent**：
- 点击「添加 Agent」按钮
- 输入名称、选择图标、设置路径
- 保存到数据库

#### 2.2.2 启用/禁用 Skill

**启用**：
- 在 skill 卡片上，点击 agent 名称旁的开关
- 将该 skill 文件夹**整体复制**到对应 agent 的 skill 路径

**禁用**：
- 关闭开关
- **自动删除** agent 文件夹中的该 skill（弹出确认对话框）

#### 2.2.3 启动时自动扫描

- 程序启动时，检查所有已启用 agent 的 skill 路径
- 发现新 skill（中央库中没有的），自动导入中央库
- 发现缺失 skill（中央库有但 agent 路径中没有的），更新数据库状态

│  │ [☑]      │  │ [ ]      │  │ [ ]      │          │

#### 2.3.2 卡片大小切换

**大卡片**（默认）：
- 显示名称、描述（前 50 字）、频率等级、标签、agent 开关
- 卡片尺寸：约 250x200px

**小卡片**：
- 只显示名称 + 图标
- 卡片尺寸：约 120x80px
- 适合 skill 数量多时的快速浏览

#### 2.3.3 侧边栏预览

- 点击 skill 卡片（不是点击开关或复选框），右侧滑出侧边栏
- 侧边栏显示：
  - Skill 名称
  - 完整描述
  - SKILL.md 的渲染内容（Markdown 格式）
  - 标签列表
  - 启用的 Agent 列表
  - 操作按钮：[编辑] [删除] [导出]
- 点击侧边栏外的区域或「关闭」按钮，侧边栏收起

#### 2.3.4 搜索

**搜索范围**：
- Skill 名称
- 标签

**搜索交互**：
- 输入关键词后实时过滤（不需要点搜索按钮）
- 支持多关键词（空格分隔，AND 关系）
- 搜索结果为空时显示提示：「未找到匹配的 skill」

#### 2.3.5 标签筛选

- 点击标签云中的标签，筛选使用该标签的 skill
- 再次点击取消筛选
- 支持多标签筛选（OR 关系）

#### 2.3.6 标签云

- 显示所有标签
- 标签大小根据使用频率动态调整（可选）
- 点击「管理标签」，弹出对话框：
  - 重命名标签
  - 合并标签（将两个标签合并为一个）
  - 删除标签（删除前确认，该标签从所有 skill 移除）

---

### 2.4 标签管理

#### 2.4.1 自动识别标签

**识别依据**：
- 解析 SKILL.md 内容，提取关键词
- 关键词来源：
  - 标题（# 一级标题、## 二级标题）
  - 触发词列表
  - 描述段落中的高频词

**识别时机**：
- 导入 skill 时自动识别
- 点击 skill 卡片上的「重新识别标签」按钮

#### 2.4.2 手动修改标签

- 在 skill 卡片或侧边栏中，点击「编辑标签」
- 弹出标签编辑对话框：
  - 显示当前标签（可删除）
  - 输入框（输入新标签，回车确认）
  - 支持多标签

---

### 2.5 频率管理

#### 2.5.1 频率等级

| 等级 | 显示 | 说明 |
|------|------|------|
| 1 | ■ 常用 | 每周使用 |
| 2 | ■ 一般 | 偶尔使用 |
| 3 | ■ 很少 | 很少使用 |
| 4 | ■ 废弃 | 不再使用，建议卸载 |

#### 2.5.2 设置频率

- 在 skill 卡片上，点击频率等级方框直接切换
- 也可以在侧边栏中修改

#### 2.5.3 按频率排序

- 主界面默认按「常用」→「一般」→「很少」→「废弃」排序
- 可以在设置中修改为按名称、按导入时间、按最后使用时间排序

---

### 2.6 批量操作

#### 2.6.1 批量选择

- 卡片左上角显示复选框
- 点击复选框选中/取消选中
- 选中后，顶部显示批量操作按钮

#### 2.6.2 全选/反选

- 「全选」— 选中所有当前显示的 skill（考虑筛选条件）
- 「反选」— 反转当前选择状态

#### 2.6.3 批量操作

- **批量启用** — 选择 agent，批量启用选中的 skill
- **批量禁用** — 批量禁用选中的 skill（从 agent 文件夹删除）
- **批量删除** — 从中央库删除选中的 skill（同时删除 agent 文件夹中的副本）
- **批量打标签** — 为选中的 skill 添加/移除标签

### 2.8 主题

## 3. 非功能需求

### 3.1 性能

- 启动时间 < 3 秒（100 个 skill 的情况下）
- 搜索响应时间 < 0.5 秒
- 复制 skill 到 agent 文件夹的进度条显示

### 3.2 兼容性

- 操作系统：Windows 10/11（优先），Linux（可选），macOS（可选）
- Qt 版本：Qt 6.5 或更高

### 3.3 可扩展性

- 支持通过配置文件添加新 agent（不需要修改代码）
- 数据库表结构设计考虑未来扩展
